# 404错误解决方案

## 问题描述

前端显示错误：`获取数据失败：Request failed with status code 404`

## 问题原因

404错误是因为数据库中还没有对应宿舍号的电费记录数据。当调用 `/api/power/records/101/latest` 时，如果数据库中没有宿舍101的记录，后端会返回404状态码。

## 解决方案

### ✅ 已完成的修复

1. **创建了测试数据**
   - 已为宿舍101创建了30条测试记录
   - 时间范围：最近30天
   - 余额范围：6.5元 到 50.0元

2. **改进了前端错误处理**
   - 当API返回404时（没有数据），不再显示错误提示
   - 显示友好的"暂无数据"提示

### 验证修复

现在可以：
1. 刷新前端页面 http://localhost:3000
2. 应该能看到电费数据和图表

### 如果仍然看到404错误

#### 方法1：检查数据是否存在

运行以下命令检查数据：

```powershell
cd c:\GitCloneRepository\dorm-power-guard-lite\backend
python create_test_data.py
```

#### 方法2：手动创建数据

如果需要为其他宿舍号创建数据，可以修改 `create_test_data.py` 中的 `dorm_number` 变量。

#### 方法3：通过API创建数据

```powershell
# 创建一条电费记录
$body = @{
    dorm_number = "101"
    balance = 50.0
    power_consumption = 5.0
    record_time = (Get-Date).ToString("yyyy-MM-ddTHH:mm:ss")
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:8000/api/power/records" -Method Post -Body $body -ContentType "application/json"
```

## 测试API端点

### 获取最新记录
```powershell
Invoke-RestMethod -Uri "http://localhost:8000/api/power/records/101/latest"
```

### 获取记录列表
```powershell
Invoke-RestMethod -Uri "http://localhost:8000/api/power/records/101?limit=10"
```
