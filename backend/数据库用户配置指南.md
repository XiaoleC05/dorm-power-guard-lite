# 数据库用户配置指南

## 问题诊断

当前错误：`Access denied for user 'cxldatabase'@'localhost'`

**原因：** MySQL 用户 `cxldatabase` 不存在或没有访问 `dorm_power_guard` 数据库的权限。

---

## 解决方案

### 方法一：创建新用户并授权（推荐）

#### 步骤 1：使用 root 用户登录 MySQL

```powershell
mysql -u root -p
```

输入您的 root 密码。

#### 步骤 2：执行以下 SQL 命令

```sql
-- 创建用户
CREATE USER 'cxldatabase'@'localhost' IDENTIFIED BY '783688';

-- 授予权限（确保数据库已创建）
GRANT ALL PRIVILEGES ON dorm_power_guard.* TO 'cxldatabase'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;

-- 验证用户创建
SELECT User, Host FROM mysql.user WHERE User='cxldatabase';

-- 退出
EXIT;
```

#### 步骤 3：验证连接

```powershell
cd c:\GitCloneRepository\dorm-power-guard-lite\backend
python test_db_connection.py
```

---

### 方法二：使用 root 用户（快速方案）

如果您想直接使用 root 用户，可以修改 `.env` 文件：

```env
DB_USER=root
DB_PASSWORD=您的root密码
```

---

### 方法三：使用 SQL 脚本（自动化）

我已经创建了 `create_user.sql` 脚本，您可以：

```powershell
# 方法1：直接执行SQL文件
mysql -u root -p < backend\create_user.sql

# 方法2：登录MySQL后执行
mysql -u root -p
source backend\create_user.sql
```

## 常见问题

### Q: 如果提示 "CREATE USER IF NOT EXISTS" 语法错误？

**A:** 某些 MySQL 版本不支持 `IF NOT EXISTS`，使用：

```sql
CREATE USER 'cxldatabase'@'localhost' IDENTIFIED BY '783688';
```

如果用户已存在，会提示错误，可以忽略。

### Q: 如何检查用户是否已创建？

```sql
SELECT User, Host FROM mysql.user WHERE User='cxldatabase';
```

### Q: 如何检查用户权限？

```sql
SHOW GRANTS FOR 'cxldatabase'@'localhost';
```
