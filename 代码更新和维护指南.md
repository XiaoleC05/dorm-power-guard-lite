# 代码更新和维护指南

## 概述

将项目部署到云服务器后，后续代码修改和更新非常方便。本指南提供多种更新方式，**推荐使用Git进行版本控制**，这是最方便、最安全的方式。

---

## 方案一：使用Git更新（推荐）

### 优点
- ✅ **版本控制**：可以回退到任意版本
- ✅ **代码同步**：本地和服务器代码保持一致
- ✅ **更新简单**：一条命令即可更新
- ✅ **安全性高**：不会丢失代码
- ✅ **协作方便**：多人协作更容易

### 1. 初始化Git仓库（如果还没有）

#### 在本地项目目录：

```bash
cd c:\GitCloneRepository\dorm-power-guard-lite
git init
git add .
git commit -m "Initial commit"
```

#### 创建远程仓库（推荐使用GitHub/Gitee）

**GitHub**：
1. 访问 https://github.com
2. 创建新仓库（设为私有）
3. 按照提示推送代码

**Gitee（国内更快）**：
1. 访问 https://gitee.com
2. 创建新仓库（设为私有）
3. 按照提示推送代码

```bash
# 添加远程仓库
git remote add origin https://github.com/your-username/dorm-power-guard-lite.git
# 或
git remote add origin https://gitee.com/your-username/dorm-power-guard-lite.git

# 推送代码
git push -u origin master
```

### 2. 在服务器上克隆代码

```bash
# SSH连接到服务器
ssh root@your_server_ip

# 克隆代码
cd /opt
git clone https://github.com/your-username/dorm-power-guard-lite.git
# 或
git clone https://gitee.com/your-username/dorm-power-guard-lite.git

cd dorm-power-guard-lite
```

### 3. 后续代码更新流程

#### 方式A：本地修改后推送到服务器（推荐）

**步骤1：在本地修改代码**

```bash
# 在本地项目目录
cd c:\GitCloneRepository\dorm-power-guard-lite

# 修改代码...

# 提交更改
git add .
git commit -m "描述你的修改"
git push origin master
```

**步骤2：在服务器上拉取更新**

```bash
# SSH连接到服务器
ssh root@your_server_ip

# 进入项目目录
cd /opt/dorm-power-guard-lite

# 拉取最新代码
git pull origin master

# 更新Python依赖（如果有新依赖）
cd backend
pip3 install -r requirements.txt

# 重启服务
sudo systemctl restart dorm-power-guard
```

#### 方式B：直接在服务器上修改（不推荐，但可行）

```bash
# SSH连接到服务器
ssh root@your_server_ip
cd /opt/dorm-power-guard-lite

# 修改代码（使用vim/nano）
nano backend/app/services.py

# 提交更改
git add .
git commit -m "服务器上直接修改"
git push origin master

# 重启服务
sudo systemctl restart dorm-power-guard
```

### 4. 创建自动化更新脚本

创建更新脚本 `update.sh`：

```bash
#!/bin/bash
# 代码更新脚本

set -e

echo "=========================================="
echo "开始更新代码..."
echo "=========================================="

# 进入项目目录
cd /opt/dorm-power-guard-lite

# 备份当前代码
echo "1. 备份当前代码..."
BACKUP_DIR="/opt/backups/code"
mkdir -p $BACKUP_DIR
tar -czf $BACKUP_DIR/backup_$(date +%Y%m%d_%H%M%S).tar.gz .

# 拉取最新代码
echo "2. 拉取最新代码..."
git fetch origin
git pull origin master

# 更新Python依赖
echo "3. 更新Python依赖..."
cd backend
pip3 install -r requirements.txt

# 运行数据库迁移（如果有）
if [ -f "apply_migration.py" ]; then
    echo "4. 运行数据库迁移..."
    python3 apply_migration.py
fi

# 重启服务
echo "5. 重启服务..."
sudo systemctl restart dorm-power-guard

# 等待服务启动
sleep 3

# 检查服务状态
if systemctl is-active --quiet dorm-power-guard; then
    echo "=========================================="
    echo "更新成功！服务已重启"
    echo "=========================================="
else
    echo "=========================================="
    echo "警告：服务可能未正常启动，请检查日志"
    echo "查看日志: sudo journalctl -u dorm-power-guard -n 50"
    echo "=========================================="
fi
```

赋予执行权限：

```bash
chmod +x update.sh
```

使用方式：

```bash
# 在本地推送代码后，在服务器上运行
./update.sh
```

## 方案二：使用rsync同步（适合频繁小修改）

### 优点
- ✅ 快速同步
- ✅ 只传输修改的文件
- ✅ 可以排除不需要的文件

### 配置rsync同步

#### 在本地创建同步脚本 `sync_to_server.sh`：

```bash
#!/bin/bash
# 同步代码到服务器

SERVER="root@your_server_ip"
REMOTE_DIR="/opt/dorm-power-guard-lite"
LOCAL_DIR="."

# 排除不需要同步的文件
EXCLUDE="--exclude='.git' --exclude='node_modules' --exclude='__pycache__' --exclude='*.pyc' --exclude='.env' --exclude='logs'"

# 同步代码
rsync -avz --delete $EXCLUDE $LOCAL_DIR/ $SERVER:$REMOTE_DIR/

# 在服务器上重启服务
ssh $SERVER "cd $REMOTE_DIR/backend && pip3 install -r requirements.txt && sudo systemctl restart dorm-power-guard"

echo "同步完成！"
```

#### Windows PowerShell版本 `sync_to_server.ps1`：

```powershell
# 同步代码到服务器（Windows）

$server = "root@your_server_ip"
$remoteDir = "/opt/dorm-power-guard-lite"
$localDir = "."

# 使用rsync（需要安装Git Bash或WSL）
# 或使用WinSCP、FileZilla等工具手动同步

Write-Host "请使用以下方式同步：" -ForegroundColor Yellow
Write-Host "1. 使用Git推送代码，然后在服务器上git pull" -ForegroundColor Green
Write-Host "2. 使用WinSCP等工具手动上传修改的文件" -ForegroundColor Green
Write-Host "3. 使用VS Code的Remote SSH扩展" -ForegroundColor Green
```

## 方案三：使用VS Code Remote SSH（推荐给开发者）

### 优点
- ✅ **直接在服务器上编辑**：就像编辑本地文件
- ✅ **实时同步**：修改立即生效
- ✅ **调试方便**：可以直接调试服务器上的代码
- ✅ **终端集成**：内置SSH终端

### 配置步骤

1. **安装VS Code扩展**
   - 打开VS Code
   - 搜索并安装 "Remote - SSH" 扩展

2. **配置SSH连接**
   - 按 `F1` 或 `Ctrl+Shift+P`
   - 输入 "Remote-SSH: Connect to Host"
   - 选择 "Configure SSH Hosts"
   - 编辑 `~/.ssh/config` 文件：

```
Host myserver
    HostName your_server_ip
    User root
    Port 22
    IdentityFile ~/.ssh/id_rsa
```

3. **连接到服务器**
   - 按 `F1`
   - 输入 "Remote-SSH: Connect to Host"
   - 选择 "myserver"
   - 输入密码（或使用SSH密钥）

4. **打开远程文件夹**
   - 连接到服务器后
   - File -> Open Folder
   - 选择 `/opt/dorm-power-guard-lite`

5. **编辑和调试**
   - 就像编辑本地文件一样
   - 修改后保存
   - 在终端中重启服务：`sudo systemctl restart dorm-power-guard`

## 方案四：使用FTP/SFTP工具（适合非开发者）

### 推荐工具

- **WinSCP**（Windows）：https://winscp.net
- **FileZilla**（跨平台）：https://filezilla-project.org
- **MobaXterm**（Windows）：https://mobaxterm.mobatek.net

### 使用步骤

1. **下载并安装工具**
2. **配置连接**
   - 主机：服务器IP
   - 端口：22（SSH）
   - 用户名：root
   - 密码：服务器密码

3. **上传修改的文件**
   - 连接到服务器
   - 导航到 `/opt/dorm-power-guard-lite`
   - 上传修改的文件（覆盖原文件）

4. **重启服务**
   - 使用SSH终端连接服务器
   - 运行：`sudo systemctl restart dorm-power-guard`

## 不同场景的更新策略

### 场景1：日常小修改（修改配置、修复bug）

**推荐方式**：Git推送 + 服务器拉取

```bash
# 本地
git add .
git commit -m "修复xxx bug"
git push

# 服务器
cd /opt/dorm-power-guard-lite
git pull
sudo systemctl restart dorm-power-guard
```

### 场景2：频繁调试和测试

**推荐方式**：VS Code Remote SSH

- 直接在服务器上编辑
- 保存后立即重启服务测试
- 调试完成后提交到Git

### 场景3：重大更新（添加新功能）

**推荐方式**：Git分支 + 测试

```bash
# 创建新分支
git checkout -b feature/new-feature

# 开发新功能
# ...

# 提交并推送
git add .
git commit -m "添加新功能"
git push origin feature/new-feature

# 在服务器上测试
ssh server
cd /opt/dorm-power-guard-lite
git fetch origin
git checkout feature/new-feature
# 测试...

# 测试通过后合并到主分支
git checkout master
git merge feature/new-feature
git push origin master
```

### 场景4：紧急修复（生产环境bug）

**推荐方式**：快速修复 + 回滚准备

```bash
# 1. 创建备份
cd /opt/dorm-power-guard-lite
git tag backup-$(date +%Y%m%d-%H%M%S)

# 2. 快速修复（直接在服务器上或本地）
# ...

# 3. 提交修复
git add .
git commit -m "紧急修复：xxx"
git push

# 4. 如果修复失败，快速回滚
git reset --hard backup-20260211-120000
sudo systemctl restart dorm-power-guard
```

## 最佳实践

### 1. 使用Git进行版本控制

```bash
# 初始化Git（如果还没有）
git init
git add .
git commit -m "Initial commit"

# 创建远程仓库（GitHub/Gitee）
# 推送代码
git remote add origin https://github.com/your-username/dorm-power-guard-lite.git
git push -u origin master
```

### 2. 创建更新脚本

使用上面提供的 `update.sh` 脚本，一键更新。

### 3. 配置自动备份

```bash
# 添加到crontab
crontab -e

# 每天凌晨3点备份代码
0 3 * * * cd /opt/dorm-power-guard-lite && git tag backup-$(date +\%Y\%m\%d) && git push origin --tags
```

### 4. 使用环境变量管理配置

**不要将 `.env` 文件提交到Git**：

```bash
# .gitignore
backend/.env
frontend/.env
*.env
```

在服务器上单独配置 `.env` 文件。

### 5. 使用Docker（可选，更高级）

如果使用Docker部署，更新更简单：

```bash
# 拉取最新代码
git pull

# 重新构建并重启
docker-compose down
docker-compose up -d --build
```

## 常见问题

### Q1: 更新代码后服务无法启动怎么办？

**A**: 
```bash
# 查看错误日志
sudo journalctl -u dorm-power-guard -n 50

# 检查Python语法错误
python3 -m py_compile backend/app/*.py

# 回滚到上一个版本
cd /opt/dorm-power-guard-lite
git log  # 查看提交历史
git reset --hard <commit-hash>  # 回滚到指定版本
sudo systemctl restart dorm-power-guard
```

### Q2: 如何避免更新时丢失配置？

**A**: 
- 使用 `.env` 文件管理配置（不提交到Git）
- 使用 `config.example` 作为配置模板
- 重要配置使用环境变量

### Q3: 多人协作时如何避免冲突？

**A**: 
- 使用Git分支开发
- 定期拉取最新代码：`git pull`
- 提交前先拉取：`git pull && git push`
- 使用 `.gitignore` 排除个人配置文件

### Q4: 如何快速回滚到之前的版本？

**A**: 
```bash
cd /opt/dorm-power-guard-lite
git log  # 查看提交历史
git reset --hard <commit-hash>  # 回滚到指定版本
sudo systemctl restart dorm-power-guard
```

### Q5: 更新时需要停止服务吗？

**A**: 
- **小修改**：可以直接更新，然后重启服务
- **重大更新**：建议先停止服务，更新后再启动
- **数据库迁移**：必须停止服务，执行迁移后再启动

## 总结

**推荐工作流程**：

1. ✅ **使用Git进行版本控制**（必须）
2. ✅ **本地开发，推送到远程仓库**
3. ✅ **服务器上拉取更新**
4. ✅ **使用更新脚本自动化流程**
5. ✅ **重要更新前先备份**

**更新代码非常方便**，使用Git后，通常只需要：
- 本地：`git push`
- 服务器：`git pull && sudo systemctl restart dorm-power-guard`

整个过程不到1分钟！
