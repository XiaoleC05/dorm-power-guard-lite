# 功能更新说明 - 余量和用电量

## ✅ 更新内容

### 1. 术语变更

**变更：**
- "余额" → "余量"
- 单位：从"元"改为"度"

**影响范围：**
- 数据库字段注释
- API响应
- 前端显示
- 告警规则

### 2. 用电量计算

**新增功能：**
- 空调用电量（kpower_consumption）：当前空调余量 - 上次空调余量
- 照明用电量（zpower_consumption）：当前照明余量 - 上次照明余量

**计算逻辑：**
- 自动计算每次记录与上次记录的差值
- 如果差值为负（可能是充值），设为NULL
- 仅在保存新记录时计算

### 3. 告警规则更新

**变更：**
- 单个阈值 → 分别设置空调和照明阈值
- 单位：从"元"改为"度"

**新增字段：**
- `kthreshold`：空调告警阈值（度）
- `zthreshold`：照明告警阈值（度）
- `threshold`：保留兼容性，已废弃

**告警逻辑：**
- 分别检查空调余量和照明余量
- 分别触发告警
- 告警日志记录告警类别（ac/light）

## 📝 数据库变更

### power_records 表

**新增字段：**
- `kpower_consumption` FLOAT NULL - 空调用电量（度）
- `zpower_consumption` FLOAT NULL - 照明用电量（度）

**字段注释更新：**
- `kbalance` → 空调余量（度）
- `zbalance` → 照明余量（度）
- `balance` → 电费余量（度）

### alert_rules 表

**新增字段：**
- `kthreshold` FLOAT NULL - 空调告警阈值（度）
- `zthreshold` FLOAT NULL - 照明告警阈值（度）

**字段变更：**
- `threshold` → 可空，已废弃

**数据迁移：**
- 旧的 `threshold` 值自动迁移到 `kthreshold`

### alert_logs 表

**新增字段：**
- `alert_category` VARCHAR(20) NULL - 告警类别（ac/light）

**字段注释更新：**
- `balance` → 触发告警时的余量（度）
- `threshold` → 告警阈值（度）

## ⚠️ 注意事项

### 数据兼容性

**旧数据：**
- 旧的 `threshold` 值已迁移到 `kthreshold`
- 旧的记录没有用电量数据（首次记录）
- 旧的告警日志没有类别信息

### 用电量计算

**首次记录：**
- 没有上次记录，用电量为NULL

**充值情况：**
- 如果余量增加（差值为负），用电量设为NULL
- 表示可能是充值，不是用电

**正常用电：**
- 用电量为正数
- 表示消耗的电量

### 告警逻辑

**分别检查：**
- 空调余量 < 空调阈值 → 触发空调告警
- 照明余量 < 照明阈值 → 触发照明告警
- 可以分别配置不同的阈值

**兼容性：**
- 如果只设置了旧的 `threshold`，会作为空调阈值使用
- 建议迁移到新的 `kthreshold` 和 `zthreshold`
