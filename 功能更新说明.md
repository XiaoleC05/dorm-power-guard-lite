# 功能更新说明

## ✅ 已实现的功能

### 1. 启动时自动获取余额

**功能描述：**
- 每次启动后端服务时，自动执行一次数据抓取
- 确保服务启动后立即有最新的余额数据

**实现位置：**
- `backend/app/main.py` - 使用 `lifespan` 上下文管理器在启动时执行

**工作流程：**
```
应用启动
  ↓
执行初始数据抓取
  ↓
初始化定时任务调度器
  ↓
服务就绪
```

### 2. 手动重新获取功能

**功能描述：**
- 在前端监控面板添加"重新获取"按钮
- 点击后调用后端API重新获取最新余额数据
- 获取成功后自动刷新显示

**实现位置：**
- 后端：`backend/app/api/system.py` - `/api/system/crawl` 接口
- 前端：`frontend/src/views/Dashboard.vue` - "重新获取"按钮

**使用方式：**
1. 打开前端监控面板
2. 点击"重新获取"按钮
3. 等待数据获取完成（显示加载状态）
4. 自动刷新显示最新数据

### 3. 刷新显示功能（保留）

**功能描述：**
- 保留原有的"刷新显示"按钮
- 仅刷新前端显示，不从后端重新获取数据
- 用于快速刷新页面显示

**区别：**
- **重新获取**：调用后端API重新抓取数据，然后刷新显示
- **刷新显示**：仅刷新前端显示，不重新抓取数据

### 4. 定时刷新功能（保留）

**功能描述：**
- 保留原有的定时任务功能
- 在配置的时间点自动执行数据抓取
- 默认时间：8:00, 12:00, 18:00, 22:00

**配置位置：**
- `backend/.env` - `SCHEDULER_HOURS` 配置项

## 🎯 使用说明

### 启动服务

**后端启动：**
```bash
cd backend
python run.py
```

启动时会自动执行一次数据抓取，日志中会显示：
```
应用启动，开始执行初始数据抓取...
启动时数据抓取成功
```

### 前端操作

**监控面板功能：**

1. **重新获取按钮（绿色）**
   - 点击后调用后端API重新获取数据
   - 显示加载状态
   - 成功后自动刷新显示
   - 显示成功/失败提示

2. **刷新显示按钮（蓝色）**
   - 点击后仅刷新前端显示
   - 不重新获取数据
   - 用于快速刷新页面

### 定时任务

定时任务会在配置的时间点自动执行：
- 默认：每天 8:00, 12:00, 18:00, 22:00
- 可在 `.env` 文件中修改 `SCHEDULER_HOURS`

## 🔄 数据获取流程

### 启动时自动获取

```
1. 后端服务启动
   ↓
2. 执行初始数据抓取
   ↓
3. 保存到数据库
   ↓
4. 初始化定时任务
   ↓
5. 服务就绪
```

### 手动重新获取

```
1. 用户点击"重新获取"按钮
   ↓
2. 前端调用 POST /api/system/crawl
   ↓
3. 后端执行数据抓取
   ↓
4. 保存到数据库
   ↓
5. 返回成功/失败消息
   ↓
6. 前端显示提示并刷新显示
```

### 定时自动获取

```
1. 到达配置的时间点
   ↓
2. 定时任务触发
   ↓
3. 执行数据抓取
   ↓
4. 保存到数据库
   ↓
5. 检查告警规则
```
