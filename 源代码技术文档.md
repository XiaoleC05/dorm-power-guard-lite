## 核心模块详解

### 1. 爬虫模块 (`crawler.py`)

#### 功能说明

负责从西华大学一卡通系统抓取电费数据。

#### 核心类：`PowerCrawler`

```python
class PowerCrawler:
    """电费爬虫类 - 西华大学电费查询系统"""
```

#### 关键方法

1. **`__init__(auto_refresh_auth=True)`**
   - 初始化爬虫实例
   - 设置请求头和Cookie
   - 自动获取认证信息（如果启用）

2. **`fetch_power_data()`**
   - 抓取电费数据
   - 返回格式化的数据字典
   - 处理API响应和错误

3. **`login()`**
   - 验证认证信息有效性
   - 如果失效，自动刷新

#### 技术要点

- **请求头模拟**：模拟微信小程序的User-Agent和请求头
- **认证管理**：集成`AuthManager`自动获取和刷新认证信息
- **错误处理**：完善的异常处理和日志记录
- **数据解析**：解析JSON响应，提取电费数据

#### 代码示例

```python
def fetch_power_data(self) -> Optional[Dict]:
    """抓取电费数据"""
    try:
        # 确保认证有效
        self.login()
        
        # 构建请求参数
        params = {
            'openid': self.openid,
            'roomid': self.room_id,
            # ... 其他参数
        }
        
        # 发送请求
        response = self.session.get(url, params=params, verify=False)
        
        # 解析响应
        data = response.json()
        return {
            'dorm_number': self.dorm_number,
            'kbalance': data.get('kbalance'),
            'zbalance': data.get('zbalance'),
            # ...
        }
    except Exception as e:
        logger.error(f"抓取数据失败: {e}")
        return None
```

### 2. 认证管理模块 (`auth_manager.py`)

#### 功能说明

管理西华大学一卡通系统的认证信息（openid和JSESSIONID）。

#### 核心类：`AuthManager`

```python
class AuthManager:
    """认证管理器 - 自动获取和刷新认证信息"""
```

#### 关键方法

1. **`auto_get_auth()`**
   - 自动获取认证信息
   - 模拟小程序首次访问流程
   - 返回openid和JSESSIONID

2. **`verify_auth(openid, jsessionid)`**
   - 验证认证信息是否有效
   - 使用`queryBuildingList` API测试

3. **`refresh_jsessionid()`**
   - 刷新JSESSIONID
   - 更新配置文件

#### 技术要点

- **自动获取**：无需手动抓包，自动获取认证信息
- **验证机制**：通过API调用验证认证信息有效性
- **持久化**：将认证信息保存到配置文件

### 3. 业务逻辑层 (`services.py`)

#### 功能说明

实现核心业务逻辑，包括数据存储、告警检查等。

#### 核心类

1. **`PowerRecordService`**
   - 电费记录管理
   - 计算用电量差值
   - 查询历史记录

2. **`AlertRuleService`**
   - 告警规则管理
   - CRUD操作

3. **`AlertLogService`**
   - 告警日志管理
   - 记录告警历史

4. **`CrawlerService`**
   - 爬虫任务调度
   - 告警检查逻辑

#### 关键方法

**`CrawlerService.crawl_and_save(db, force_alert=False)`**
- 执行爬虫任务
- 保存数据到数据库
- 触发告警检查
- `force_alert`参数：手动触发时忽略防频繁告警限制

**`CrawlerService.check_and_alert(db, dorm_number, kbalance, zbalance, force_alert=False)`**
- 检查余量是否低于阈值
- 分别检查空调和照明
- 触发告警发送

**`CrawlerService._send_alert(...)`**
- 发送告警
- 防频繁告警机制（1小时间隔）
- 失败重试机制

#### 技术要点

- **用电量计算**：通过差值计算用电量，处理充值情况
- **防频繁告警**：1小时内不重复告警（手动触发除外）
- **失败重试**：上次告警失败时立即重试
- **分类告警**：空调和照明分别设置阈值和告警

### 4. 告警模块 (`alert.py`)

#### 功能说明

实现邮件和QQ机器人告警功能。

#### 核心类

1. **`EmailAlert`**
   - 邮件告警
   - 使用SMTP发送邮件
   - 支持HTML和纯文本格式

2. **`QQBotAlert`**
   - QQ机器人告警

3. **`AlertManager`**
   - 告警管理器
   - 统一管理多种告警方式

#### 关键方法

**`EmailAlert.send(...)`**
- 发送邮件告警
- 使用模板生成邮件内容
- 支持多个接收邮箱

**`QQBotAlert.send(...)`**
- 发送QQ消息
- 支持群消息和私聊

#### 技术要点

- **邮件模板**：使用HTML模板生成美观的邮件
- **SMTP配置**：支持SSL和STARTTLS
- **错误处理**：完善的异常处理和日志记录

### 5. 定时任务模块 (`scheduler.py`)

#### 功能说明

使用APScheduler实现定时任务调度。

#### 核心功能

1. **`init_scheduler()`**
   - 初始化调度器
   - 配置定时任务
   - 默认时间点：8:00, 12:00, 18:00, 22:00

2. **`shutdown_scheduler()`**
   - 关闭调度器
   - 清理资源

#### 技术要点

- **Cron表达式**：使用cron表达式配置执行时间
- **后台执行**：使用BackgroundScheduler在后台运行
- **异常处理**：任务执行异常不影响调度器运行

## 关键技术实现

### 1. 认证信息自动获取

**问题**：如何自动获取openid和JSESSIONID？

**解决方案**：
- 使用`AuthManager`模拟小程序首次访问
- 通过抓包分析API流程
- 自动获取并保存认证信息

**代码位置**：`backend/app/auth_manager.py`

### 2. 用电量计算

**问题**：如何计算用电量？

**解决方案**：
- 计算当前值与上次值的差值
- 如果差值为负（可能是充值），设为None
- 分别计算空调和照明用电量

**代码位置**：`backend/app/services.py` → `PowerRecordService.create_record()`

### 3. 防频繁告警

**问题**：如何避免频繁告警？

**解决方案**：
- 记录最后告警时间
- 1小时内不重复告警
- 手动触发时忽略限制（`force_alert=True`）
- 上次告警失败时立即重试

**代码位置**：`backend/app/services.py` → `CrawlerService._send_alert()`

### 4. 邮件模板

**问题**：如何生成美观的邮件？

**解决方案**：
- 使用HTML模板
- 支持HTML和纯文本两种格式
- 响应式设计，适配各种邮件客户端

**代码位置**：`backend/app/templates.py`

### 5. 异步任务调度

**问题**：如何实现定时任务？

**解决方案**：
- 使用APScheduler
- BackgroundScheduler在后台运行
- Cron表达式配置执行时间

**代码位置**：`backend/app/scheduler.py`

## 性能优化建议

1. **数据库索引**：在`dorm_number`和`record_time`字段上创建索引
2. **连接池**：使用SQLAlchemy连接池管理数据库连接
3. **缓存**：对频繁查询的数据使用缓存
4. **异步处理**：使用FastAPI的异步特性提高并发性能
5. **批量操作**：批量插入数据时使用批量操作

本项目采用前后端分离架构，使用FastAPI和Vue 3构建，实现了电费数据的自动抓取、存储、监控和告警功能。核心特点：

1. **模块化设计**：各模块职责清晰，易于维护

3. **可扩展性**：易于添加新的告警方式、数据源等
4. **用户友好**：提供Web界面，操作简单
