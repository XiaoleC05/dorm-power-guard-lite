# 程序持续运行部署指南

## 概述

要让程序在个人电脑关闭后也能一直运行，需要将程序部署到一台**24小时运行的服务器**上。本指南提供多种部署方案。

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **云服务器（推荐）** | 稳定可靠、24小时运行、有公网IP | 需要付费（通常很便宜） | 长期使用、需要稳定运行 |
| **本地服务器/树莓派** | 一次性投入、数据本地 | 需要硬件、需要网络配置 | 有闲置设备、对数据安全要求高 |
| **Windows服务** | 无需额外硬件 | 电脑必须24小时运行 | 有24小时运行的电脑 |
| **Docker部署** | 易于部署、环境隔离 | 需要Docker环境 | 熟悉Docker的用户 |

## 方案一：云服务器部署（推荐）

### 1. 选择合适的云服务器

**国内云服务商**：
- **阿里云**：https://www.aliyun.com
- **腾讯云**：https://cloud.tencent.com
- **华为云**：https://www.huaweicloud.com
- **百度云**：https://cloud.baidu.com

**国外云服务商**：
- **AWS**：https://aws.amazon.com
- **DigitalOcean**：https://www.digitalocean.com
- **Vultr**：https://www.vultr.com

**推荐配置**：
- CPU：1核
- 内存：1GB（2GB更佳）
- 硬盘：20GB SSD
- 操作系统：Ubuntu 20.04/22.04 LTS 或 CentOS 7/8
- 价格：通常每月 ¥20-50（国内）或 $5-10（国外）

### 2. 服务器环境准备

#### 2.1 连接到服务器

使用SSH连接到服务器：

```bash
ssh root@your_server_ip
```

#### 2.2 更新系统

```bash
# Ubuntu/Debian
apt update && apt upgrade -y

# CentOS/RHEL
yum update -y
```

#### 2.3 安装必要软件

```bash
# Ubuntu/Debian
apt install -y python3 python3-pip git mysql-server nginx

# CentOS/RHEL
yum install -y python3 python3-pip git mariadb-server nginx
```

#### 2.4 安装MySQL

```bash
# Ubuntu/Debian
apt install -y mysql-server
systemctl start mysql
systemctl enable mysql

# CentOS/RHEL
yum install -y mariadb-server
systemctl start mariadb
systemctl enable mariadb
```

配置MySQL：

```bash
mysql_secure_installation
```

创建数据库和用户：

```bash
mysql -u root -p
```

```sql
CREATE DATABASE dorm_power_guard CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'dorm_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON dorm_power_guard.* TO 'dorm_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 3. 部署程序

#### 3.1 克隆代码

```bash
cd /opt
git clone https://github.com/your-username/dorm-power-guard-lite.git
cd dorm-power-guard-lite
```

或者直接上传代码到服务器：

```bash
# 在本地打包
cd /path/to/dorm-power-guard-lite
tar -czf dorm-power-guard-lite.tar.gz .

# 上传到服务器（使用scp）
scp dorm-power-guard-lite.tar.gz root@your_server_ip:/opt/

# 在服务器上解压
ssh root@your_server_ip
cd /opt
tar -xzf dorm-power-guard-lite.tar.gz
```

#### 3.2 配置后端

```bash
cd /opt/dorm-power-guard-lite/backend

# 安装Python依赖
pip3 install -r requirements.txt

# 配置环境变量
cp .env.example .env
nano .env
```

编辑 `.env` 文件，配置数据库和爬虫信息：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=dorm_user
DB_PASSWORD=your_password
DB_NAME=dorm_power_guard

# 其他配置...
```

#### 3.3 初始化数据库

```bash
# 运行数据库迁移脚本（如果有）
python3 apply_migration.py

# 或手动执行SQL
mysql -u dorm_user -p dorm_power_guard < migrate_add_power_fields.sql
```

#### 3.4 配置前端

```bash
cd /opt/dorm-power-guard-lite/frontend

# 安装依赖
npm install

# 构建生产版本
npm run build
```

### 4. 使用 systemd 管理后端服务

创建服务文件：

```bash
sudo nano /etc/systemd/system/dorm-power-guard.service
```

内容：

```ini
[Unit]
Description=Dorm Power Guard Backend Service
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/dorm-power-guard-lite/backend
Environment="PATH=/usr/bin:/usr/local/bin"
ExecStart=/usr/bin/python3 /opt/dorm-power-guard-lite/backend/run.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable dorm-power-guard
sudo systemctl start dorm-power-guard
sudo systemctl status dorm-power-guard
```

### 5. 配置 Nginx 反向代理

创建Nginx配置：

```bash
sudo nano /etc/nginx/sites-available/dorm-power-guard
```

内容：

```nginx
server {
    listen 80;
    server_name your_domain.com;  # 或使用IP地址

    # 前端静态文件
    location / {
        root /opt/dorm-power-guard-lite/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端API
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/dorm-power-guard /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 6. 配置防火墙

```bash
# Ubuntu/Debian (ufw)
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# CentOS/RHEL (firewalld)
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### 7. 配置SSL证书（可选，推荐）

使用Let's Encrypt免费SSL证书：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your_domain.com
```

## 方案二：Windows服务部署

如果您的电脑需要24小时运行，可以将程序注册为Windows服务。

### 1. 安装 NSSM（Non-Sucking Service Manager）

下载NSSM：https://nssm.cc/download

解压后，将 `nssm.exe` 复制到系统PATH或程序目录。

### 2. 创建Windows服务

以管理员身份运行PowerShell：

```powershell
# 安装后端服务
nssm install DormPowerGuardBackend "C:\Python\python.exe" "C:\GitCloneRepository\dorm-power-guard-lite\backend\run.py"
nssm set DormPowerGuardBackend AppDirectory "C:\GitCloneRepository\dorm-power-guard-lite\backend"
nssm set DormPowerGuardBackend DisplayName "宿舍电费监控系统-后端"
nssm set DormPowerGuardBackend Description "宿舍电费监控系统后端服务"
nssm set DormPowerGuardBackend Start SERVICE_AUTO_START
nssm set DormPowerGuardBackend AppStdout "C:\GitCloneRepository\dorm-power-guard-lite\backend\logs\stdout.log"
nssm set DormPowerGuardBackend AppStderr "C:\GitCloneRepository\dorm-power-guard-lite\backend\logs\stderr.log"

# 启动服务
nssm start DormPowerGuardBackend
```

### 3. 管理服务

```powershell
# 启动服务
nssm start DormPowerGuardBackend

# 停止服务
nssm stop DormPowerGuardBackend

# 重启服务
nssm restart DormPowerGuardBackend

# 查看服务状态
nssm status DormPowerGuardBackend

# 删除服务
nssm remove DormPowerGuardBackend confirm
```

## 方案三：Docker部署

### 1. 创建 Dockerfile

**后端 Dockerfile** (`backend/Dockerfile`):

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["python", "run.py"]
```

**前端 Dockerfile** (`frontend/Dockerfile`):

```dockerfile
FROM node:18-alpine as build

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
```

### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: dorm_power_guard
      MYSQL_USER: dorm_user
      MYSQL_PASSWORD: dorm_password
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

  backend:
    build: ./backend
    environment:
      DB_HOST: mysql
      DB_PORT: 3306
      DB_USER: dorm_user
      DB_PASSWORD: dorm_password
      DB_NAME: dorm_power_guard
    depends_on:
      - mysql
    ports:
      - "8000:8000"
    volumes:
      - ./backend:/app
    restart: always

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: always

volumes:
  mysql_data:
```

### 3. 部署

```bash
docker-compose up -d
```

## 方案四：树莓派/本地服务器部署

如果有一台24小时运行的树莓派或本地服务器，可以按照方案一的步骤部署，但需要注意：

1. **网络配置**：确保服务器有固定IP或使用DDNS
2. **端口转发**：如果在内网，需要在路由器上配置端口转发
3. **动态IP处理**：如果IP会变化，使用DDNS服务（如花生壳、No-IP）

## 监控和维护

### 1. 查看日志

```bash
# systemd服务日志
sudo journalctl -u dorm-power-guard -f

# 应用日志（如果配置了日志文件）
tail -f /opt/dorm-power-guard-lite/backend/logs/app.log
```

### 2. 检查服务状态

```bash
sudo systemctl status dorm-power-guard
```

### 3. 重启服务

```bash
sudo systemctl restart dorm-power-guard
```

### 4. 设置自动备份（推荐）

创建备份脚本 `backup.sh`:

```bash
#!/bin/bash
BACKUP_DIR="/opt/backups"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u dorm_user -p'your_password' dorm_power_guard > $BACKUP_DIR/db_$DATE.sql

# 备份配置文件
tar -czf $BACKUP_DIR/config_$DATE.tar.gz /opt/dorm-power-guard-lite/backend/.env

# 删除7天前的备份
find $BACKUP_DIR -type f -mtime +7 -delete
```

添加到crontab：

```bash
crontab -e
# 每天凌晨2点备份
0 2 * * * /opt/dorm-power-guard-lite/backup.sh
```

## 常见问题

### Q1: 如何选择云服务器？

**A**: 根据预算和需求：
- 预算有限：选择国内云服务商的轻量应用服务器（¥20-30/月）
- 需要海外访问：选择Vultr、DigitalOcean等（$5-10/月）
- 需要高性能：选择标准云服务器（¥50+/月）

### Q2: 如何确保数据安全？

**A**: 
1. 定期备份数据库
2. 使用强密码
3. 配置防火墙，只开放必要端口
4. 使用SSL证书加密传输
5. 定期更新系统和依赖

### Q3: 如何监控程序运行状态？

**A**:
1. 使用systemd查看服务状态
2. 配置日志监控
3. 使用监控工具（如Prometheus + Grafana）
4. 设置邮件告警（程序已支持）

### Q4: 如何更新程序？

**A**:
```bash
cd /opt/dorm-power-guard-lite
git pull
cd backend
pip3 install -r requirements.txt
sudo systemctl restart dorm-power-guard
```
