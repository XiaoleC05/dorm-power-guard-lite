# 邮件发送配置说明

## 功能说明

系统已配置为使用 **714085964@qq.com** 作为发件邮箱，发送告警邮件到告警规则中配置的接收邮箱。

## 工作流程

1. **定时获取电费余量**：系统在配置的时间点（默认：8:00, 12:00, 18:00, 22:00）自动获取电费余量
2. **检查告警阈值**：比较当前余量与告警规则中配置的阈值
3. **触发告警**：如果余量低于阈值，自动发送邮件到告警规则中配置的接收邮箱
4. **发送邮件**：使用 `714085964@qq.com` 作为发件邮箱，发送到规则中配置的接收邮箱

## 配置步骤

### 1. 获取QQ邮箱授权码

**重要**：QQ邮箱不能直接使用QQ密码，必须使用授权码！

#### 获取授权码步骤：

1. **登录QQ邮箱**
   - 访问：https://mail.qq.com
   - 使用QQ账号登录（714085964）

2. **开启SMTP服务**
   - 点击右上角"设置" → "账户"
   - 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"部分
   - 开启"POP3/SMTP服务"或"IMAP/SMTP服务"
   - 如果之前未开启，系统会要求进行身份验证（手机验证）

3. **生成授权码**
   - 在"POP3/SMTP服务"或"IMAP/SMTP服务"开启后
   - 点击"生成授权码"按钮
   - 按提示发送短信到指定号码
   - 获取16位授权码（例如：`abcdefghijklmnop`）

### 2. 配置 `.env` 文件

编辑 `backend/.env` 文件，找到邮件配置部分，将授权码填入：

```env
# 邮件配置
EMAIL_ENABLED=true
EMAIL_SMTP_HOST=smtp.qq.com
EMAIL_SMTP_PORT=465
EMAIL_SMTP_USER=714085964@qq.com
EMAIL_SMTP_PASSWORD=你的16位授权码  # ← 替换这里
EMAIL_FROM=714085964@qq.com
EMAIL_TO=
```

**示例**（假设授权码是 `abcdefghijklmnop`）：
```env
EMAIL_SMTP_PASSWORD=abcdefghijklmnop
```

### 3. 配置告警规则接收邮箱

在前端页面配置接收告警邮件的邮箱：

1. **打开前端页面**
   - 访问前端地址（通常是 http://localhost:3000）

2. **进入告警规则页面**
   - 点击左侧菜单"告警规则"

3. **编辑告警规则**
   - 点击"编辑"按钮
   - 开启"邮件告警"开关
   - 在"接收邮箱"输入框中输入接收邮箱地址
     - 单个邮箱：`user@example.com`
     - 多个邮箱：`user1@example.com,user2@example.com`
   - 设置告警阈值（空调和照明）
   - 点击"保存"

### 4. 重启后端服务

配置完成后，重启后端服务使配置生效：

```bash
# 停止当前运行的后端服务
# 然后重新启动
cd backend
python run.py
```

## 邮件发送逻辑

1. **发件邮箱**：固定使用 `714085964@qq.com`
2. **接收邮箱**：使用告警规则中配置的 `email_address`
3. **发送时机**：
   - 定时任务执行时（默认：8:00, 12:00, 18:00, 22:00）
   - 手动点击"重新获取"时
   - 程序启动时自动获取一次
4. **发送条件**：
   - 告警规则已启用（`enabled = true`）
   - 邮件告警已启用（`email_enabled = true`）
   - 已配置接收邮箱（`email_address` 不为空）
   - 余量低于阈值（`kbalance < kthreshold` 或 `zbalance < zthreshold`）
   - 距离上次告警至少1小时（避免频繁告警）

## 测试邮件发送

配置完成后，可以通过以下方式测试：

1. **降低告警阈值**：将告警阈值设置为一个较高的值（例如：100度），这样当前余量就会低于阈值
2. **手动触发**：在前端点击"重新获取"按钮
3. **等待定时任务**：等待下一个定时任务执行时间

## 常见问题

### 1. 提示"535 Login Fail. Please enter your authorization code to login"

**原因**：使用了QQ密码而不是授权码

**解决方法**：
- 确认使用的是授权码，不是QQ密码
- 重新生成授权码并更新配置

### 2. 提示"连接超时"或"无法连接到SMTP服务器"

**原因**：网络问题或端口被防火墙阻止

**解决方法**：
- 检查网络连接
- 尝试使用端口587（STARTTLS）代替465（SSL）
- 修改 `.env` 文件：`EMAIL_SMTP_PORT=587`

### 3. 邮件发送成功但收不到邮件

**原因**：邮件可能被放入垃圾邮件文件夹

**解决方法**：
- 检查垃圾邮件文件夹
- 将发件人邮箱（714085964@qq.com）添加到联系人白名单
- 检查接收邮箱的过滤规则

### 4. 提示"未配置接收邮箱地址"

**原因**：告警规则中未配置 `email_address`

**解决方法**：
- 在前端"告警规则"页面编辑规则
- 开启"邮件告警"
- 在"接收邮箱"输入框中输入邮箱地址
- 保存

## 安全建议

1. **不要将授权码提交到Git仓库**
   - 确保 `.env` 文件已添加到 `.gitignore`
   - 授权码泄露后应立即重新生成

2. **定期更换授权码**
   - 建议每3-6个月更换一次授权码

3. **限制接收邮箱**
   - 只配置必要的接收邮箱
   - 避免将告警邮件发送到公开邮箱
