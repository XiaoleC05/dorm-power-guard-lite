# 邮箱配置和测试指南

## 当前配置

- **发件邮箱**: `1270667498@qq.com`
- **SMTP服务器**: `smtp.qq.com`
- **SMTP端口**: `465` (SSL)

## 配置步骤

### 1. 获取QQ邮箱授权码

1. **登录QQ邮箱**
   - 访问：https://mail.qq.com
   - 使用QQ账号 `1270667498` 登录

2. **开启SMTP服务**
   - 点击右上角"设置" → "账户"
   - 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"部分
   - 开启"POP3/SMTP服务"或"IMAP/SMTP服务"
   - 如果之前未开启，系统会要求进行身份验证（手机验证）

3. **生成授权码**
   - 在"POP3/SMTP服务"或"IMAP/SMTP服务"开启后
   - 点击"生成授权码"按钮
   - 按提示发送短信到指定号码
   - 获取16位授权码（例如：`abcdefghijklmnop`）

### 2. 配置 `.env` 文件

编辑 `backend/.env` 文件，找到以下配置：

```env
EMAIL_SMTP_PASSWORD=请替换为你的QQ邮箱授权码
```

将 `请替换为你的QQ邮箱授权码` 替换为你获取的16位授权码，例如：

```env
EMAIL_SMTP_PASSWORD=abcdefghijklmnop
```

**完整配置示例**：
```env
EMAIL_ENABLED=true
EMAIL_SMTP_HOST=smtp.qq.com
EMAIL_SMTP_PORT=465
EMAIL_SMTP_USER=1270667498@qq.com
EMAIL_SMTP_PASSWORD=你的16位授权码
EMAIL_FROM=1270667498@qq.com
EMAIL_TO=
```

### 3. 配置告警规则接收邮箱（推荐）

在前端页面配置接收告警邮件的邮箱：

1. **打开前端页面**
   - 访问：http://localhost:3000

2. **进入告警规则页面**
   - 点击左侧菜单"告警规则"

3. **编辑告警规则**
   - 点击"编辑"按钮
   - 开启"邮件告警"开关
   - 在"接收邮箱"输入框中输入接收邮箱地址
     - 单个邮箱：`user@example.com`
     - 多个邮箱：`user1@example.com,user2@example.com`
   - 设置告警阈值（空调和照明）
   - 点击"保存"

## 常见问题

### 1. 提示"535 Login Fail. Please enter your authorization code to login"

**原因**：使用了QQ密码而不是授权码

**解决方法**：
- 确认使用的是授权码，不是QQ密码
- 重新生成授权码并更新配置

### 2. 提示"连接超时"或"无法连接到SMTP服务器"

**原因**：网络问题或端口被防火墙阻止

**解决方法**：
- 检查网络连接
- 尝试使用端口587（STARTTLS）代替465（SSL）
- 修改 `.env` 文件：`EMAIL_SMTP_PORT=587`

### 3. 邮件发送成功但收不到邮件

**原因**：邮件可能被放入垃圾邮件文件夹

**解决方法**：
- 检查垃圾邮件文件夹
- 将发件人邮箱（1270667498@qq.com）添加到联系人白名单
- 检查接收邮箱的过滤规则

### 4. 提示"未配置接收邮箱地址"

**原因**：告警规则中未配置 `email_address`

**解决方法**：
- 在前端"告警规则"页面编辑规则
- 开启"邮件告警"
- 在"接收邮箱"输入框中输入邮箱地址
- 保存

## 邮件发送逻辑

1. **发件邮箱**：固定使用 `1270667498@qq.com`
2. **接收邮箱**：使用告警规则中配置的 `email_address`
3. **发送时机**：
   - 定时任务执行时（默认：8:00, 12:00, 18:00, 22:00）
   - 手动点击"重新获取"时
   - 程序启动时自动获取一次
4. **发送条件**：
   - 告警规则已启用（`enabled = true`）
   - 邮件告警已启用（`email_enabled = true`）
   - 已配置接收邮箱（`email_address` 不为空）
   - 余量低于阈值（`kbalance < kthreshold` 或 `zbalance < zthreshold`）
   - 距离上次告警至少1小时（避免频繁告警）
