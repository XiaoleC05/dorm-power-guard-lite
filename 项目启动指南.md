# 西华大学宿舍电费监控系统 - 完整启动指南

> **项目说明**：本项目针对西华大学一卡通宿舍用电小程序，用于监控宿舍电费使用情况。  
> **管理员QQ**：714085964

## 📋 前置要求

- **Python 3.8+** 
- **Node.js 16+** 和 npm
- **MySQL 5.7+**
- **Git**（可选）

---

## 🚀 完整启动步骤

### 第一步：准备数据库

#### 1.1 启动MySQL服务

**Windows:**
```cmd
# 检查MySQL服务是否运行
net start MySQL80
# 或
net start MySQL
```

**Linux/Mac:**
```bash
sudo systemctl start mysql
# 或
sudo service mysql start
```

#### 1.2 创建数据库

```bash
mysql -u root -p
```

在MySQL命令行中执行：
```sql
CREATE DATABASE dorm_power_guard CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

---

### 第二步：配置后端

#### 2.1 进入后端目录

```bash
cd backend
```

#### 2.2 安装Python依赖

**Windows:**
```cmd
install.bat
```

**Linux/Mac:**
```bash
chmod +x install.sh
./install.sh
```

**或手动安装:**
```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

#### 2.3 配置环境变量

**Windows:**
```cmd
copy .env.example .env
```

**Linux/Mac:**
```bash
cp .env.example .env
```

**编辑 `.env` 文件**，至少修改以下配置：

```env
# 数据库配置（必须）
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=dorm_power_guard

# 爬虫配置（必须，但可以先不配置Token，后续再配置）
CRAWLER_BASE_URL=https://ecard.xhu.edu.cn
CRAWLER_API_BASE_URL=https://ecard.xhu.edu.cn/api
CRAWLER_DORM_NUMBER=101

# Token暂时留空，后续通过抓包获取
# CRAWLER_TOKEN=your_token_here
```

#### 2.4 初始化数据库表

后端启动时会自动创建表，也可以手动初始化：

```bash
python -c "from app.database import init_db; init_db()"
```

---

### 第三步：启动后端服务

#### 3.1 使用启动脚本（推荐）

**Windows (命令提示符 CMD):**
```cmd
cd backend
start.bat
```

**Windows (PowerShell):**
```powershell
cd backend
.\start.bat
# 或使用PowerShell脚本
.\start.ps1
```

**Linux/Mac:**
```bash
cd backend
chmod +x start.sh
./start.sh
```

#### 3.2 或直接运行

```bash
cd backend
python run.py
# 或
python3 run.py
```

#### 3.3 验证后端启动成功

看到以下输出表示成功：
```
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

**访问验证:**
- API文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

**保持后端运行**，打开新的终端窗口进行下一步。

---

### 第四步：配置和启动前端

#### 4.1 进入前端目录

```bash
cd frontend
```

#### 4.2 安装前端依赖



看到以下输出表示成功：
```

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

**访问前端界面：** http://localhost:3000

## 🔧 常见问题解决

### 问题1：后端启动失败 - 数据库连接错误

**错误信息：**
```
OperationalError: (2003, "Can't connect to MySQL server")
```

**解决方案：**
1. 检查MySQL服务是否启动
2. 检查 `.env` 中的数据库配置
3. 确认数据库已创建

### 问题2：后端启动失败 - 模块未找到

**错误信息：**
```
ModuleNotFoundError: No module named 'fastapi'
```

**解决方案：**
```bash
pip install -r requirements.txt
```

### 问题3：后端启动失败 - 端口被占用

**错误信息：**
```
ERROR:    [Errno 48] Address already in use
```

**解决方案：**
1. 修改 `backend/run.py` 中的端口（如改为8001）
2. 或关闭占用8000端口的程序：
   ```bash
   # Windows
   netstat -ano | findstr :8000
   taskkill /PID <进程ID> /F
   
   # Linux/Mac
   lsof -ti:8000 | xargs kill -9
   ```

### 问题4：前端启动失败 - 依赖安装失败

**解决方案：**
```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

### 问题5：前端无法连接后端

**检查：**
1. 后端是否在运行
2. `frontend/vite.config.js` 中的proxy配置是否正确
3. 浏览器控制台是否有CORS错误
